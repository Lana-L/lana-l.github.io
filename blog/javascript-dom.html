Javascript relationship to HTML and CSS

Javascript is the engine that makes the webpage run.

HTML gives us the content - the words on the page, the meaning of those words (eg, content flags, such as aside, list, heading)

CSS is the beauty technician - it controls how everything looks. Two pages with the same content could look completely different with different css styles.

Javascript makes everything interactive. Granted, some functions that look interactions can be achieved with CSS, like CSS transitions, but Javascript can count, add, multiply, can store information, and can pass information to other places.

The DOM is the document object model - it is the 'house that HTML and CSS built'. Javascript can access parts of that house, like knocking on the door and walking in, then cooking some dinner in the kitchen. Without the DOM, JS would not be able to interact with that house.

Control Flow

Control flow is the order in which the things in a program happen. For example, CSS control flow is top to bottom - so if you have a conflicting style that comes later, it'll override the earlier one.

In Javascript, functions also execute from top to bottom, but some functions can call other functions, and then return things back out to the main flow, which can end up very similar to a 'pick-a-path' book.

A loop is an example of controlling the flow - it tells the program to run a certain number of times and repeat itself until a condition is met that lets it stop the loop. For example, "eat this lolly until your teeth ache" - you bite into a lolly, teeth are ok, keep eating, you do it again, and again, and again, then suddenly your teeth ache, ah! time stop eating the lolly. The loop has ended because the exit condition has been met.

Arrays and Objects

An array is a list of items, it can be of any type (numbers, strings), and it's an iterative list - this means loops can loop through them, and we can count how many items there are in the list, and we can add more items, or remove some items, and modify the items.

Objects are bizarre beings that don't make sense - they at first look similar to arrays, but they also can have things within them that are not items, like functions, or even arrays! Objects have key-value pairs - like "name: Bob" and "price: 10". In a way, an object is like a simplified data table, and it can have many 'columns' and rows. Each 'column' can be anything, even an array or a function. Working with objects seems much more complicated than working with arrays or databases though.

Functions

Functions are the mini-programs that do things. For example, if there is a function to pour water out of a bottle, it may have instructions like hold bottle, open cap, tilt bottle, pour water, reduce the value of how much water there is in the bottle by how much was poured out, close cap, show the bottle with now less water in it.

Because these same steps are taken every time water is poured of the bottle, a function allows us to write them just once, and then every time we pour out water, we "call" the function and the computer repeats the same steps, saving us time from writing it out from scratch every time.
