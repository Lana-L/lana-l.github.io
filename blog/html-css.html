What does it mean to display inline vs inline blocks?

display:inline-block allows setting of width and height of the element, whereas display:inline does not. Although both can have padding and margins, so there is a way to get around the limitations.

Both are often used in making flat nav bars, such as the one seen at the top of this page - it's actually an unordered list that is displayed as an inline-block, whereas the nav bar on the left has no such display styling.

The entire top breadcrumbs nav bar is shown "inline", but each element is "inline-block" because then a slight padding is added to show spacing between the breadcrumbs. This could have been done with just plain text > added after each entry, but this would have underlined it as it was being styled by the a property (links). This way, the > is added programmatically with content property of CSS after the a (using a::after) and then styled to have padding, so it doesn't run into the next entry.
